# Прототип генератора заданий для виртуального лабораторного практикума по анатомии

Это Godot 4.3+ проект — прототип интерфейса преподавателя для создания шаблонов и генерации заданий по новым типам (из слайдов ВКР):

- **Идентификация кости**
- **Определение ориентиров**
- **Сборка сустава**
- **Сборка анатомической конструкции**

Генератор передаёт визуализатору (твоя часть):
- `anatomical_area_id`
- списки `bone_ids` (target + extra)
- `correct_connections` (массив пар `[id1, id2]` для сборки)

## Как запустить

1. Открой папку `anatomy_generator_godot` в **Godot 4.3+**.
2. Убедись, что в Project Settings → Autoload уже есть `GlobalData` (добавлено автоматически).
3. Главная сцена уже установлена (`scenes/Main.tscn`).
4. Нажми **Play** (F5).

## Что уже работает (прототип)

### Режим Преподавателя
- Карточки статистики (обновляются live).
- 4 кнопки быстрого создания шаблонов (по одному на каждый тип задания) с разумными параметрами.
- Список "Мои шаблоны" (сохраняется в `user://templates.json`).
- Выбор шаблона → кнопки:
  - **Сгенерировать 3 задания** по шаблону (сохраняются в `user://generated_assignments.json`).
  - **Предпросмотр примера** — показывает, какие данные уйдут в визуализатор (включая `correct_connections`).
- RichTextLabel с последними сгенерированными заданиями.

### Режим Студент (заглушка)
- Список всех сгенерированных заданий.
- При выборе — детальный разбор данных, которые получит визуализатор (id костей, правильные соединения и т.д.).
- Кнопка **"Запустить в визуализаторе (заглушка)"** — показывает сообщение, что именно передаётся 3D-части.
- Кнопка **"Завершить задание (демо)"** — имитирует результат (60–100%).

## Данные (БД)

Всё хранится в `data/anatomy_data.json` (редактируй при необходимости):
- `anatomical_areas`
- `bones` (с `landmarks` у некоторых)
- `joints` (с `bone_ids` и `correct_connections`)
- `structures` (с `bone_ids` и `correct_connections`)

Генератор в `GlobalData.gd`:
- `_generate_bone_identification`
- `_generate_landmark_identification`
- `_generate_joint_assembly`
- `_generate_structure_assembly`

## Что дальше делать (рекомендации по этапам)

1. **Сейчас** — протестируй генерацию, посмотри JSON в предпросмотре. Убедись, что данные (id + connections) подходят для твоего визуализатора.
2. **Следующий этап** — доработать полноценную форму создания/редактирования шаблона (динамические поля в зависимости от типа задания) вместо быстрых кнопок. Можно сделать отдельную сцену `CreateTemplateWindow.tscn` + скрипт.
3. **Интеграция** — когда будет готова 3D-часть визуализатора, добавь сигнал/вызов, который передаёт `assignment.data` в визуализатор (например, через Autoload или сцену).
4. **Расширение БД** — добавь больше костей, ориентиров, суставов, структур (особенно для черепа, нижней конечности). Можно сделать импорт из CSV/Excel.
5. **Сохранение результатов** — добавить таблицу результатов студента и аналитику (как в ВКР).
6. **Экспорт** — кнопка "Экспортировать задания в JSON для визуализатора".

## Известные ограничения прототипа
- Предпросмотр использует простой AcceptDialog (кнопка "Регенерировать" работает через повторный вызов).
- В генераторе структуры/суставов `extra_pool.erase` может иногда оставлять дубликаты в очень редких случаях — легко поправить фильтром.
- Нет полноценного редактирования шаблонов (только создание и генерация).
- Шрифт NotoSans указан, но если нет файла — Godot использует встроенный (поддержка кириллицы есть в 4.x).

## Файлы проекта
- `scripts/GlobalData.gd` — ядро (загрузка JSON, генератор, сохранение)
- `scripts/MainUI.gd` — весь интерфейс (динамический)
- `scenes/Main.tscn` — корневая сцена
- `data/anatomy_data.json` — "база данных" костей/суставов/структур
- `project.godot` — настроен на автозагрузку и главную сцену
